## Tests (TDD order: RED → GREEN → REFACTOR per task)

Order is encoded in `tasks.md`. High level:

1. **Library first** — types → channel-name helper → `getLeadCanvas` (unit
   with mocked client) → fixture → fixture-validates-against-leadSchema test.
2. **Renderers** — primitive renderers + `<SchemaMismatch />` + tier badge
   (pure UI, fast tests with RTL).
3. **Composite components** — header, field block, suggested-action slot,
   agent-panel slot. Each component a focused RTL test.
4. **Activity stream + realtime hook** — hook with injected mock channel;
   stream uses the hook. Cross-org filter tested in unit AND integration.
5. **Canvas root** — `<LeadCanvas>` integration RTL: section order,
   "More" toggle, reduced-motion (mock matchMedia), AI-row tier badge.
6. **Routes** — demo page renders fixture; `[id]` route uses
   `getLeadCanvas` → `notFound()` on null.
7. **Integration (Supabase)** — RLS scoping (canvas-rls) and Realtime
   isolation (canvas-realtime-isolation) against the real project.
8. **Playwright @smoke** — demo route + 404 route.
9. **Baseline 112** — ratified at end of Group D.

## Risks (for Plan Mode reviewer)

| # | Risk | Severity | Mitigation |
|---|---|---|---|
| P-1 | Framer Motion + React 19 + RSC. First motion install; could surface incompatibilities at build. | Med | Lock to a React 19-compatible major; all motion-using components are `'use client'`; build runs as a verification step. |
| P-2 | Supabase Realtime test against the live project might be flaky on CI. | Med | Use a fresh per-run channel name `canvas:lead:<test-uuid>`; `await new Promise(r => setTimeout(r, 1500))` after subscribe before insert; clean up subscriptions in `afterEach`. |
| P-3 | The Canvas reads `audit_log` to derive `agent_tier` per activity. If the join is slow, first paint slips past 1.5s. | Med | LIMIT 50, query is a single LEFT JOIN with two indexed FKs. D-014 hardens; D-006 only smoke-asserts. |
| P-4 | "More" toggle implemented with `AnimatePresence` may cause layout shift on first render. | Low | Initial state of "More" is collapsed; SSR renders only primary fields; expansion is post-hydration. |
| P-5 | Demo route at a public-ish URL (still gated by `/dashboard/*` middleware) may confuse users into thinking the data is real. | Low | "Demo lead — fictional data, no DB row" banner; copy reviewed by reviewer in Plan Mode. |
| P-6 | First Client Component tree of this size in the repo. RSC boundary discipline matters. | Low | `<LeadCanvas>` and all its descendants are `'use client'`; pages are Server Components that pass serialized props. |
| P-7 | Cross-tenant 404 vs 403 — leaks existence if we pick wrong. | Low | Always 404 (`notFound()`). Tests verify. |
| P-8 | Activity stream replay on reconnection. If the websocket drops and reconnects, missed messages aren't replayed. | Low | Out of scope for D-006; documented in baseline 112 as V1. |
| P-9 | Tier badge derivation depends on actor_type='agent' rows in audit_log existing. Until D-009 ships agents, no AI rows render. | Low | Tested via fixture (which includes one synthetic AI row); production traffic produces no AI rows in V0 until D-009. |
